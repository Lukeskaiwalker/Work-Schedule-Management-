# v2.3.5 - Hotfix: in-UI update installer was broken since v2.3.0 (git "dubious ownership")

Date: 2026-05-01

## What was broken

Clicking "Update installieren" in the admin updates panel always failed
with:

```
fatal: detected dubious ownership in repository at '/repo'
To add an exception for this directory, call:
    git config --global --add safe.directory /repo
[... ] Finished. status=failed exit_code=128
```

The bug has been latent since v2.3.0 (when the runner-mediated update
flow was introduced) but was never exercised from the UI. Operators
were running `safe_update.sh` directly from the host shell, where git
runs as the host user `mac` and the ownership check passes.

## Root cause

Git's CVE-2022-24765 protection refuses to operate on a repository
owned by a different uid than the running process. In the runner
sidecar:

- Process: root (uid 0, default for the python:3.12-alpine image)
- `/repo` ownership: bind-mounted from host `/home/mac/SMPL-all`,
  owned by `mac` (uid 1000)

Mismatch → "dubious ownership" → exit 128 before any of the script's
real work can run.

## Fix

One line in `apps/update_runner/Dockerfile`:

```dockerfile
RUN git config --global --add safe.directory /repo
```

Tells git to trust `/repo` regardless of file ownership. Localized to
the runner image (root user's `~/.gitconfig`) so it doesn't touch any
host or other-container config. Safe because this image's only purpose
is operating on `/repo`.

## Compatibility

- Same `safe_update.sh --pull --branch main` rollout flow.
- The v2.3.3 `--force-recreate update_runner` step ensures the new
  runner image (with the safe.directory line baked in) actually replaces
  the running container.
- No new env vars, no new permissions, no DB changes.

## Verification

After rollout, click "Update installieren" in admin → updates. The
runner should now successfully run `git fetch + git pull` and proceed
through the rest of the safe_update flow. The audit log will show
`runner.install_started` followed by progress events instead of the
failed-immediately pattern.
