# v1.3.0 - Chat feed, Office task controls, and update safety hardening

Date: 2026-03-03

## Highlights

- Persistent report-feed chat:
  - Adds/maintains a global `Latest Construction Reports` thread.
  - Automatically posts each new construction report PDF with project number and project name in the message body.
  - Backfills missing report-feed entries and prevents deleting the system feed thread.
- Overview latest reports:
  - Adds `GET /api/construction-reports/recent` and a latest-10 reports block on the overview page.
- Task workflow improvements:
  - Tasks support optional due dates (`due_date = null`).
  - Adds server-derived `is_overdue` flag so overdue tasks can be highlighted/filtered reliably.
- Office/Construction workflow UX updates:
  - Workspace mode switch in the sidebar header (`Construction` / `Office`).
  - Office task filters extended (status, assignee, date, project search, no-due-date handling).
- Release metadata display fixes:
  - Improves installed-version resolution in update status and user menu popup (no placeholder-only `local-production` label when resolvable).
- Safer backup and restore operations:
  - `backup`, `restore`, and migration preflight scripts now use stream-copy instead of `docker compose cp` and enforce secure temp-directory permissions.

## Upgrade Notes

- This release introduces no new Alembic migration files.
- Recommended update path:
  - `BACKUP_PASSPHRASE='<passphrase>' ./scripts/safe_update.sh --pull --branch main`
- Standard manual path:
  - `docker compose up -d --build`
  - `docker compose run --rm api alembic upgrade head`
- Never run `docker compose down -v` during standard upgrades.

## Validation Baseline

- `./scripts/test.sh` passes on this release (`61 passed`) and frontend production build completes successfully.
