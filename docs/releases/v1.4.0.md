# v1.4.0 - Materials catalog expansion, office message overview, and release metadata automation

Date: 2026-03-05

## Highlights

- Materials workflow expansion:
  - DATANORM import/parser alignment for electrical catalog data.
  - Material catalog search in side menu with result limit control and project search UX improvements.
  - Catalog item image enrichment via EAN lookups (manufacturer-first, open EAN fallback) with duplicate-import skip reporting.
- Project/report UX updates:
  - Construction report office notes are surfaced in project overview (office view only).
  - Task/report material-entry behavior fixed so commas do not split a single typed line unexpectedly.
  - Known item IDs in task/report text can resolve against catalog data for consistent material entries.
- Data-entry reliability updates:
  - Task edit modal close behavior hardened for mouse-selection edge case.
  - Time-input fields normalize to `HH:MM` with automatic zero-padding on blur.
- Release/version operations hardening:
  - Added `scripts/update_release_metadata.sh` to generate runtime release metadata (`apps/api/.release.env`) from git tag/commit.
  - `scripts/safe_update.sh` now refreshes this metadata automatically before rebuild/deploy.
  - Compose now loads `.release.env` for `api` and `api_worker`, preventing stale hardcoded version labels.

## Upgrade Notes

- Standard update path (recommended):
  - `BACKUP_PASSPHRASE='<passphrase>' ./scripts/safe_update.sh --pull --branch main`
- Material catalog sync on running server:
  - catalog files from `./Datanorm_Neuanlage` are mounted at `/data/Datanorm_Neuanlage`,
  - import runs automatically on catalog access and can be verified with `/api/materials/catalog/state`.
- Never run `docker compose down -v` during normal upgrades.
