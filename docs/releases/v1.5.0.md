# v1.5.0 - Responsive workspace, project site addresses, and material status visibility

Date: 2026-03-09

## Highlights

- Added dual-address project support end-to-end:
  - new `construction_site_address` field in project model, schemas, API update/create paths, and CSV/XLS import mapping,
  - map/weather now prefer construction site address and fall back to customer address,
  - project overview contact card now shows both customer and construction site addresses.
- Completed mobile/tablet responsive workspace pass:
  - collapsible off-canvas sidebar with hamburger toggle, overlay, and auto-close navigation behavior,
  - improved small-screen layouts for planning, messages/chat, admin/profile tables, and key workspace sections,
  - viewport height usage moved to dynamic units (`dvh`) to reduce mobile browser clipping.
- Extended material catalog image-import status telemetry in UI/API:
  - added lookup phase visibility and fallback-queue counters,
  - improved progress diagnostics in Materials view.
- Included recent platform improvements since `v1.4.1`:
  - backend material image lookup moved off request path,
  - full frontend page lazy-loading and chunk splitting,
  - realtime SSE updates and task notifications,
  - backend router/model/schema/test refactors with updated domain coverage.

## Upgrade Notes

- Apply DB migrations before serving traffic:
  - `docker compose run --rm api sh -lc "cd /app && alembic upgrade head"`
- Recommended local/network rollout:
  - `./scripts/safe_update.sh --pull --branch main`

## Validation Baseline

- `./scripts/test.sh` passes (`79 passed` backend tests + web production build pass).
- `docker compose up -d --build api web caddy` starts healthy services locally.
