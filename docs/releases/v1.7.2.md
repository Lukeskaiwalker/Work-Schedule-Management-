# v1.7.2 - Legacy attachment decrypt regression fix

Date: 2026-03-17

## Highlights

- Fixed a `v1.7.1` regression in encrypted file validation for legacy Fernet-stored attachments.
- File preview/download for older attachments now works again instead of failing with:
  - `Stored file payload cannot be decrypted with current key`
- Added regression coverage for legacy attachment validation so mixed storage formats stay supported.

## Root Cause

- Chunked `SMPLENC2` attachment storage was introduced on 2026-02-25.
- Attachments stored before that change still use the older Fernet payload format.
- `v1.7.1` added pre-stream validation, but the validation fallback path did not rewind the file handle before Fernet decrypt.
- That truncated the first 8 bytes of older payloads and caused false decryption failures even when the key was correct.

## Upgrade Notes

- No new Alembic migrations are introduced after `v1.7.0`.
- Recommended rollout:
  - `BACKUP_PASSPHRASE='<passphrase>' ./scripts/safe_update.sh --pull --branch main`

- `./scripts/test.sh` passes (`81 passed` backend tests + web production build pass).
